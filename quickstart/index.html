Last Chance Guide

快速开始

真的不复杂。复杂的是总有人喜欢把“填 URL 和 key”演成一场软件祭祀。

你需要准备什么

  • cc-switch
  • 先搞清楚你要用的是 `paolu` 还是 `luo`，这两个站不是同一个站。
  • 你自己的 API Key，格式示例：sk-XXXXXXXXXXXXXXXXXXXXXXXX

三步接入

第一步：下载 cc-switch

先去 GitHub 下载。兵器都没领，别急着问剑招。

第二步：新建自定义配置

在 cc-switch 里新建一个自定义配置。这里最容易翻车：两个网站不一样，URL 也不一样。

第三步：填 key 然后打开 Codex

把 API Key 填进去，格式示例就是 sk-XXXXXXXXXXXXXXXXXXXXXXXX。保存，重新打开 Codex。到这里就够了，别再给流程加支线任务。

Do Not Mix Them Up

这两个网站不一样，填错就会报错

很多人都错在这里：`paolu` 和 `luo` 看着像一家的，但你在 `cc-switch` 里填的是具体某一个站的 API URL。不要把 A 站的 key 和 B 站的 URL 混着填。

如果你用 `paolu` 站

站点入口：https://paolu.hudawang.cn/

https://paolu.hudawang.cn/v1

如果你用 `luo` 站

站点入口：https://luo.hudawang.cn/

https://luo.hudawang.cn/v1

API Key 示例

key 格式就长这样，别把示例里的 X 当成真实密钥。

sk-XXXXXXXXXXXXXXXXXXXXXXXX

cc-switch 里大概就填成这样

箭头指的就是最容易填错的两个地方：上面是 sk-... key，下面是你选中的站点对应 URL。

顺手给你放三个入口

https://paolu.hudawang.cn/

主站入口。大多数人真正要进去配置和使用的就是这里。

https://luo.hudawang.cn/

另一套站点入口。需要的时候直接点，不需要的时候也别把它和主站搞混。

https://status.hudawang.cn/

状态页。怀疑服务抽风之前，先来这里看看到底是谁先跑路了。

如果你非要测一下

curl https://paolu.hudawang.cn/v1/models \
  -H "Authorization: Bearer sk-XXXXXXXXXXXXXXXXXXXXXXXX"

能拿到模型列表，说明路没堵死。拿不到，先检查是不是 URL 或 key 抄错了。
